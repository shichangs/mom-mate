# UI/UX 方向 02：情感成长感（品牌优先）

## 1. 设计目标

- 让产品从“工具”升级为“陪伴型成长记录”。
- 强化里程碑和每日总结的情绪价值，提升记忆点。
- 在不增加操作复杂度的前提下，提升视觉品质和品牌一致性。

## 2. 核心体验原则

1. 先情绪，再数据：先让用户感受到“宝宝今天状态”，再看详细统计。
2. 单屏单任务：每屏只强调一个主操作（记录入睡、添加饮食、记录里程碑）。
3. 轻仪式感：关键动作（完成睡眠、添加里程碑）提供明确反馈动画。
4. 一致性第一：颜色、圆角、阴影、按钮层级只保留一套规范。

## 3. 视觉系统（建议）

### 3.1 主题语义色

- `sleep`：#6B7BFF（夜间安定）
- `meal`：#35C57A（营养成长）
- `milestone`：#FF9F66（喜悦时刻）
- `insight`：#4DA3FF（数据洞察）
- `danger`：#FF5A5F（删除/警告）

### 3.2 背景与材质

- 主背景：浅暖渐变（米白到淡粉）  
- 卡片背景：半透明高亮卡 + 细描边（弱玻璃感）
- 避免过重阴影，采用 2 层轻阴影表达层级

### 3.3 字体层级

- `Hero`：32/700（首页情绪标题）
- `Title`：24/700
- `Section`：18/600
- `Body`：16/400
- `Meta`：13/400

### 3.4 圆角与间距

- 卡片：20
- 按钮：16
- Chip：999
- 间距基线：8pt 网格（8/16/24/32）

## 4. 交互与动效规范

### 4.1 动效

- 页面首次进入：卡片 120ms 渐入 + 轻微上移
- 主按钮点击：0.92 scale 按压反馈
- 完成记录：成功状态 400ms 徽章动画（不过度）

### 4.2 反馈

- 所有新增/删除操作必须有 toast 或 inline 反馈
- 删除提供撤销（短时 undo）

### 4.3 空状态

- 每个 Tab 统一空状态模板：插画 + 一句关怀文案 + 一个 CTA

## 5. 页面改造重点（按优先级）

## P0：睡眠首页（本项目第一阶段重点）

- 顶部改为“今日状态条”：今日累计睡眠 + 与昨日对比
- 中部保留唯一主卡：醒着/睡眠中状态切换
- 底部“最近记录”收敛为 3 条 + 查看全部入口
- “选择其他时间”改成二级按钮，不与主按钮争抢视觉权重

## P0：统计页

- 首屏只放 3 张洞察卡：今日/本周/趋势
- 图表颜色统一到语义色，减少混色
- 每张图表下补一句解释文案（系统可生成）

## P1：里程碑页

- 强化时间轴感：日期锚点 + 时刻卡片
- 新增成功后提供“庆祝反馈”小动效
- 快捷添加改为“常见里程碑胶囊流”

## P1：饮食页

- 新增入口改为底部悬浮主按钮（更稳定）
- 常用食物与自定义食物同一面板，支持最近使用优先

## 6. 内容文案方向（示例）

- 睡眠空状态：`今晚从第一条记录开始，宝宝的好梦值得被记住。`
- 里程碑完成提示：`又一个第一次，已为你保存。`
- 统计摘要：`这周睡眠比上周更稳定，继续保持。`

## 7. 开发落地任务拆分（两周冲刺）

### Sprint A（第 1 周）

1. 建立新 design token（颜色/字体/圆角/间距）
2. 重构睡眠首页主卡 + 状态条
3. 重构统计页首屏信息层级
4. 完成空状态模板组件

### Sprint B（第 2 周）

1. 里程碑页时间轴改版
2. 饮食页新增流程收敛
3. 补全动效与反馈（toast/undo）
4. 全量页面视觉一致性走查

## 8. 验收标准

1. 用户 10 秒内可完成一次核心记录（睡眠/饮食/里程碑）
2. 四个 Tab 的视觉元素一致（色彩、按钮、卡片、间距）
3. 空状态、错误状态、成功反馈都可见且统一
4. 动态字体放大到 1.5x 不破版
5. 首屏交互路径不超过 2 层点击

## 9. 风险与边界

- 风险：视觉表达过强会影响信息可读性  
  约束：所有装饰优先级低于数据和操作

- 风险：动画过多降低性能  
  约束：单次页面动效不超过 2 处，同时总时长 < 500ms
